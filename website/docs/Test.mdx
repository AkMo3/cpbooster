`cpbooster` provides the following command for you to easily test your program against
all available test cases.

```txt
cpb test <filePath>
```

or the alias for `test` which is just `t`.

```txt
cpb t <filePath>
```


or you can specify the test case, using the `--testId` flag or its alias `-t`.

```txt
cpb t <filePath> -t <number>
```

## How does it work?

If your code needs compilation, this will be the first thing that `cpbooster` will
try to do, if the compilation fails it will print something similar to this:

IMAGE HERE

If the compilation is successful or your code didn't need any compilation it will continue in
this way, it is going to find all the `<fileName>.inX` files and use them as input
for your program, it will then compare your program's output with the corresponding
`<fileName>.ansX` files, 


:::important
Notice that `<fileName>` should also be the name of the corresponding source file (`cpp`, `py`, ...),
since `cpbooster` uses `<fileName>` to make associations between source files and test cases. See [File Structure](/docs/clone#file-structure)
:::
