`cpbooster` provides the following command for you to easily test any program.

```txt
cpb test <filePath> [options]
```

This command will test your program against all available test cases (default behavior)
or just one test case if `--testId` flag is specified. It supports the following veredicts.

- <span style={{background: colors.green, color: "white"}} >&nbsp;<b>AC</b>&nbsp;</span> (Accepted Solution)
- <span style={{background: colors.red, color: "white"}}>&nbsp;<b>WA</b>&nbsp;</span> (Wrong Answer)
- <span style={{background: colors.yellow, color: "white"}}>&nbsp;<b>CE</b>&nbsp;</span> (Compilation Error)
- <span style={{background: colors.purple, color: "white"}}>&nbsp;<b>TLE</b>&nbsp;</span> (Time Limit Exceeded)
- <span style={{background: colors.blue, color: "white"}}>&nbsp;<b>RTE</b>&nbsp;</span> (Runtime Error)

## How does it work?

If your code needs compilation, this will be the first thing that `cpbooster` will
try to do, if the compilation fails it will print something similar to this:

IMAGE HERE

If the compilation is successful or your code didn't need any compilation it will continue in
this way, it is going to find all the `<fileName>.inX` files and use them as input
for your program, it will then compare your program's output with the corresponding
`<fileName>.ansX` files, 


:::important
Notice that `<fileName>` should also be the name of the corresponding source file (`cpp`, `py`, ...),
since `cpbooster` uses `<fileName>` to make associations between source files and test cases. See [File Structure](/docs/clone#file-structure)
:::
